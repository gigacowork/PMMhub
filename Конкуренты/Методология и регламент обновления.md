## 2. Шкала оценок heatmap

- 🟢 G — возможность есть, подтверждена документацией или анонсом GA
- 🟡 A — частично: бета, ограниченный сценарий, через обходной путь (API/MCP)
- 🔴 R — отсутствует или явно не поддерживается
- ⚪ N — нет данных (не путать с R; N — повод для проверки в следующий обход)

## 3. Правила достоверности данных

1. Каждый новый факт — с датой и источником. Источник пишется в лист «Журнал обновлений».
2. Иерархия источников: (1) официальные сайты/доки вендора → (2) пресс-релизы и крупные СМИ (CNews, Ведомости, Коммерсантъ, Habr-блоги компаний, VentureBeat, BusinessWire) → (3) агрегаторы и обзоры — только с пометкой «⚠️ сверить».
3. Цены в $ пересчитываются по фиксированному курсу (сейчас 1 $ = 80 ₽); курс указывается в дашборде и пересматривается раз в квартал.
4. Обновления существующих ячеек xlsx — добавлением блока `[Обновлено ДД.ММ.ГГГГ]: …`, без удаления истории.
5. Не подтверждённое двумя независимыми источниками событие высокой значимости помечается «⚠️».

### Рекомендации по моделям Anthropic

| Роль | Модель | Почему |
|---|---|---|
| Еженедельный обход и обновление файлов | **Claude Sonnet 4.6** | Оптимум цена/качество для поиска, верификации и аккуратных правок двух файлов. Рекомендуемый дефолт |
| Массовый сбор (если обходов станет много: ежедневный мониторинг, десятки источников) | Claude Haiku 4.5 | В разы дешевле по токенам; годится для «собрать ссылки и черновик фактов», финальную запись в базу лучше отдавать Sonnet |
| Ежемесячная глубокая ревизия (пересмотр методологии, скоринг, новые секции, редизайн дашборда) | Claude Opus 4.8 / Fable 5 (текущая сессия) | Максимальное качество рассуждений для структурных решений; дорого для еженедельной рутины |

Практический паттерн: scheduled-задача работает на модели по умолчанию (Sonnet); раз в месяц запускайте ручную сессию глубокой ревизии на старшей модели по разделу 4 этого документа.
